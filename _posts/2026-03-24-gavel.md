Look at code snippet from `includes/bid_watcher.php`. What this code is doing is that when we place a bid, it creates a function at runtime using `runkit_function_add`. The rule set for a bid is placed directly into function's body without sanitization.

```
try {
    if (function_exists('ruleCheck')) {
        runkit_function_remove('ruleCheck');
    }
    runkit_function_add('ruleCheck', '$current_bid, $previous_bid, $bidder', $rule);
    error_log("Rule: " . $rule);
    $allowed = ruleCheck($current_bid, $previous_bid, $bidder);
} catch (Throwable $e) {
    error_log("Rule error: " . $e->getMessage());
    $allowed = false;
}
```

Using the admin panel we set a reverse shell payload as rule, then we will use a normal user session to execute payload by placing the bid.
