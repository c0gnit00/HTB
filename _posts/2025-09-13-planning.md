## Executive Summary

Planning is an easy-difficulty Linux machine on HackTheBox that focuses on subdomain discovery, exploiting a critical SQL Injection vulnerability in Grafana, extracting credentials from Docker environment variables, and compromising a locally hosted cron management service.

The compromise begins with initial credentials for the `admin` user. Subdomain enumeration discovers a Grafana instance at `grafana.planning.htb`. The Grafana service is vulnerable to CVE-2024-9264, a post-authorization SQL Injection flaw utilizing DuckDB. Exploiting this vulnerability via a Python script executes a reverse shell, granting access to a Docker container running the Grafana service.

Inside the container, inspection of environment variables reveals the administrative SSH credentials for the host user `enzo`. Authenticating as `enzo` via SSH provides access to the host system and the user flag.

For privilege escalation, local port scanning reveals a service running on localhost port 8000/tcp, which manages scheduled cron tasks. Investigating the `/opt/crontabs` directory uncovers a database file (`crontab.db`) that leaks a backup password. Port forwarding port 8000 and logging in to the web panel using the credentials `root:P4ssw0rdS0pRi0T3c` allows the attacker to create and run a new cron job. Because the cron job runner executes with root privileges, running a reverse shell script achieves full system compromise.

As is common in real life pentests, you will start the Planning box with credentials for the following account: admin / 0D5oT70Fq13EvB5r

### CVE-2024-9264 — Grafana DuckDB SQL Injection

The Grafana version running on this target is vulnerable to **CVE-2024-9264**, a post-authentication SQL injection flaw in Grafana's DuckDB data source integration. The vulnerability works as follows:

1. **DuckDB as a vector**: Grafana supports DuckDB as a data source for querying. DuckDB is an in-process SQL OLAP database that includes powerful functions like `read_blob()` (for arbitrary file reads) and `read_csv_auto()` (for reading local files as tables).

2. **SQL Injection via API**: An authenticated user can craft a malicious DuckDB SQL query through Grafana's API. Since Grafana does not properly sanitize user-supplied input before passing it to DuckDB, an attacker can inject arbitrary DuckDB SQL statements.

3. **File Read → RCE**: The PoC exploit abuses DuckDB's `read_blob()` function to read arbitrary files from the server (e.g., `/etc/passwd`). For RCE, the exploit passes a base64-encoded command that gets decoded and executed in a shell — effectively chaining the SQL injection with a command execution through DuckDB's file-reading capabilities into a Docker container shell.

## Mitigations & Security Recommendations

1. **Patch and Upgrade Grafana**:
   - Update the Grafana service to a secure version that mitigates CVE-2024-9264 (which restricts unauthorized DuckDB queries and file reads).
   - Harden DuckDB features by disabling external file execution or restricting query capabilities.

2. **Secure Container Environment Secrets**:
   - Avoid passing sensitive parameters like administrative passwords (`GF_SECURITY_ADMIN_PASSWORD`) as environment variables.
   - Employ secure vault solutions (e.g., HashiCorp Vault) or container orchestration secret mounts to inject credentials during runtime.

3. **Protect Local Configuration Files and Databases**:
   - Restrict permissions on configuration directories (such as `/opt/crontabs/`). Ensure that the `crontab.db` database is readable only by the owner/root process, and prevent standard users (like `enzo`) from reading configuration secrets.

4. **Harden the Local Task Scheduling Application**:
   - Secure the web panel running on port 8000. Do not use shared or default passwords (e.g., `P4ssw0rdS0pRi0T3c`).
   - If the task runner must execute cron jobs, configure it to run commands under lower-privileged user contexts rather than executing them directly as `root`. Implement strict input validation or code signing to prevent users from executing arbitrary shell commands.
